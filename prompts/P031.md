P031: Add topic label editing (ARCHITECTURE.md decision 33)

Allow users to rename topics from auto-generated labels (e.g. "Topic 1
(ada, augusta, guido)") to custom labels (e.g. "Family & Marriage").
Custom labels should propagate to ALL charts and exports.

1. IN src/lemmata/app.py --- add label editing UI:

- In the sidebar, after analysis completes, add a new expander: "Edit
topic labels"

- Inside: one text_input per topic, pre-filled with current
auto-generated label

- A "Apply labels" button that updates st.session_state with custom
labels

- When custom labels exist, ALL tab functions should use them instead
of auto-generated labels

- Store custom labels in st.session_state["custom_topic_labels"] as
a list[str]

- Add a "Reset to default" link/button to revert to auto-generated
labels

2. IN src/lemmata/app.py --- propagate custom labels:

- Create a helper function get_topic_labels(results,
custom_labels=None) -> list[str]

- If custom_labels exist in session_state and have the right length,
use them

- Otherwise fall back to results["topic_labels"]

- Update ALL \_tab\_* functions to use get_topic_labels() instead of
directly reading results["topic_labels"]

- This affects: \_tab_overview, \_tab_topics, \_tab_topic_map,
\_tab_heatmap, \_tab_distribution, \_tab_preprocessing, \_tab_export

3. IN src/lemmata/file_io.py --- propagate to exports:

- The export functions (ZIP, PDF, CSV) should accept topic_labels
parameter

- When custom labels exist, exported files use them

4. Tests:

- Test get_topic_labels with and without custom labels

- Test that custom labels of wrong length are ignored (fallback to
default)

Run tests: pytest -m "not slow" --tb=short -q

Commit as: "P031: Add topic label editing with propagation to all
charts and exports"
