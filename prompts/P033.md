```
P033: Add file preview, empty file handling, duplicate file warning, and
estimated analysis time

Four UX improvements for the pre-analysis stage (after file upload,
before "Run Analysis"):

1. FILE PREVIEW (ARCHITECTURE.md decision 65):

After files are uploaded (or text is pasted), show a preview section
with:

- For each file: filename, file size (KB/MB), estimated word count,
first 200 characters of content

- Use st.expander per file, collapsed by default, with a summary line
showing filename + size + word count

- For pasted text: show character count and first 200 characters

- This helps users verify they uploaded the correct files before
running analysis

2. EMPTY FILE HANDLING (ARCHITECTURE.md decision 64):

- If a file is empty (0 bytes or only whitespace after cleaning), skip
it silently

- Show st.warning: "Skipped {filename}: file is empty or contains only
whitespace."

- Continue processing remaining files normally

- If ALL files are empty, show st.error and do not run analysis

- Test edge case: file with only newlines/spaces

3. DUPLICATE FILE WARNING (ARCHITECTURE.md decision 163):

- After upload, check for files with identical names

- If duplicates found, show st.warning: "Duplicate filename detected:
{filename}. Both copies will be included --- remove one if this was
unintentional."

- Also check for files with identical content (even if different
names): compare first 500 chars + file size

- Non-blocking warning, do not prevent analysis

4. ESTIMATED ANALYSIS TIME (ARCHITECTURE.md decision 161):

- After files are loaded and before analysis starts, estimate time:

Simple formula: base_time (2s) + (n_documents * n_topics * 0.05s) +
(total_words / 10000 * 0.5s)

- Show st.info: "Estimated analysis time: ~{seconds}s" (round to
nearest 5s)

- If estimated > 60s, add: "This is a large corpus. Consider reducing
chunk size or topic count for faster results."

- Show this in the sidebar or main area, just above the progress
indicator

Implementation notes:

- All features go in the pre-analysis stage (after upload, before Run)

- File preview should work for all 5 supported formats (txt, pdf, docx,
odt, epub)

- Add tests for: empty file skip, duplicate detection, time estimation
formula

- Do not break existing 147 tests

- Run: pytest -m "not slow" --tb=short -q
```
