## Fix 1: CRITICAL --- Crash on small/mismatched corpus

The app crashes completely (Streamlit "Oh no" error) when a very short
text is pasted or when the wrong language model is selected (e.g.,
Italian model with English text). The root cause is an unhandled
exception when CountVectorizer produces an empty or too-small DTM.

In app.py, wrap the entire analysis pipeline (from preprocessing through
modelling) in a try/except that catches ALL exceptions. On failure:

- Show st.error with a user-friendly message: "Analysis failed. This
usually happens when the text is too short, the wrong language is
selected, or too few words remain after preprocessing. Try uploading a
larger text or checking your language setting."

- Show the technical error in a st.expander("Technical details")

- Do NOT let the exception propagate to Streamlit's global error
handler

- Make sure session state is cleaned up properly so the app remains
usable after the error

## Fix 2: Language mismatch warning

In preprocessing.py, after spaCy processing is complete, count how many
tokens were successfully lemmatized vs total tokens. If the ratio of
kept tokens to total tokens is below 10%, add a warning to the
preprocessing trace dict with key "language_warning": "Only X% of
tokens were recognized. You may have selected the wrong language."

In app.py, after preprocessing completes, check for this warning and
display it as st.warning BEFORE proceeding to modelling. The warning
should say: "Low token recognition rate (X%). This may indicate a
language mismatch. Check that the selected language matches your text."

- The chart actually renders data points, not just empty axes
